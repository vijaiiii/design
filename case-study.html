‣ To design a responsive ‘game review’ section for Gaming Website, in order to help its users to browse and read the review of games

Project Goal:

‣ To design a responsive ‘game review’ section for Gaming Website, in order to help its users to browse and read the review of games.

Assumptions and Hypothesis:

‣ Right now, there is no place where a user can find all the games details at one place.
‣ There is no place where a design user can see in detail reviews about all the games.
‣ There is no place where a user can decide where to start with depending on his interests and knowledge level.

Secondary Research:

Right now no where user can find all games at single portal with details. So, there are no competitor websites/organizations for this project.

Scenarios:

1.User is new to design and no idea on the games /game publishers.
2.User is new to design and has idea on the games to play.
3.User is not new to design field. Would like to increase the current knowledge by exploring new games /game publishers through reviews.

KPI (Key Performance Indicators):

‣Bounce rate of ‘Game Review’ page.
‣ How much time people have spent on the page
‣ How many people write the game review.
‣ Search engine (Eg: Google, Bing etc) traffic to the “Game Review” section.

Establishing Project Goals:

Overall research findings (goals and pain points of both business and users) were defined to establish project goals.

Project Goals

Product Road map

A feature road map was developed using inputs from secondary research and interviews. Must have MVP features requested by participants as essential to a review website were given high priority. Typical features in a game review website was added along with the way which would help users in their game selection process. A competitor feature matrix was done for indirect brand competitors (since there is no direct competitors). It helped to analyze the cost and value of implementing features and services to find which will be profitable and easy to execute with the most gains; identify which features are available/not available in competitors website.

Testing

Usability Test

To determine the level of efficiency of design and usability within the user interface of the interactive prototype, remote usability testings were conducted. The test plan was devised were to understand user preference, memory and identify potential navigation issues within the task flow.
From the usability test findings, a list was prepared using findings in navigation & usability; content and call to action, suggestions from participants and recommended course of action.
From the UT results, it can be deduced that the prototype is success for the defined workflows.

Observations:

‣Users were very happy to know about a portal, which has only reviews for games.
‣There were no errors during all work flows performed with the new prototype.
‣Time taken for the workflows were very minimal.
‣Users most liked the ‘adding review’ flow.
‣All users expected to have total number of reviews next to reviews on game list pages.
‣Users found games list pages on mobile are too long to scroll down.

Concluding Remarks:

Some of the key learning from usability testing were:

1.Long list of items on mobile screens bothers users.
2.Users prefer easy eye catching information's in the design.

As part of redesign/improvements to the designed prototype, we considered following changes -

1.Adding total number of reviews on game list pages.
2.Adding versions of the games on game detail page.
3.Long list of items on mobile screens will redesigned for better user experience.
